May 26, 2019

An Introduction

What's up friends. This blog post begins a series of blog posts involving re-doing my personal website. The old website was statically typed, ugly, and to be honest just kind of bored me so I've decided to completely redo it from the ground up and take the opportunity to learn some interesting web technologies along the way. At the time of writing, this blog post is, and the website as a whole, are still statically typed in HTML and CSS. My intentions are to make a new website where I can add to the blog posts all from an online interface. This'll likely require some kind of database, admin interface (with login), and more! With each new feature or large step in development I'll be making one of these posts and talking about how everything's going. Lets begin!

The Toolset

A programmer is only as efficient as their toolkit (kind of). Beginning with package managers, something I'm only even mentionning because its a first for me on the Windows end of things. On MacOS I'm using the tried and true Homebrew. On Windows I'm using Chocolatey, a first for me. Now for the interesting stuff. I've decided to use React. I felt that the weight of a full framework like Angular was a bit unnecessary for the purpose of my personal website and trying to understand it would take up extra time whereas React is simply a library. Furthermore, I intend to learn and use SASS, SCSS in specific, since CSS seems to be a little bit repititious and using a pre-processor to save some time sounds dandy. Also I am going to be using SCSS rather than traditional SASS syntax since I'd like to have the ability to put pure CSS into the SASS script and don't care too much about a couple semicolons and brackets. We'll see how development goes. Since this is purely a personal project I'm really free to change, add, and remove and technologies as I please.
